Where the human changed what the AI did

  1. Absolute → relative features. Human rejected raw Hz/dB; forced semitone ratios, percentile ranks and range-normalised slopes so cues transfer across speakers and languages.
  2. Random split → GroupKFold by turn_id, scored on the real delay metric, not accuracy/AUC.
  3. English-only → cross-lingual EN→HI as the primary proxy ("hidden test is mostly Hindi"); this exposed that raw-pitch features transfer at chance.
  4. Rejected the 760 ms headline as in-sample optimism and reported the honest ~850 ms instead.
  5. Mandated the five leak checks and reviewed each pause_end access by hand.
  6. Caught two shipping bugs: the cumulative-silence features were dead (never passed prev_pause_durs), and the single tree gave unseparated probabilities (threshold 0.05). Directed the fix — revive the features and switch to an LR + ExtraTrees ensemble — which moved the honest Hindi threshold to 0.45 and AUC to 0.735.

The AI wrote code fast; the human made the judgement calls that made it correct.
